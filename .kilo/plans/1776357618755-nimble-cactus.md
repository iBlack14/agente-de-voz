# GuГӯa: ВҝQuГ© es un VPS y CГіmo Subir Tu Sistema?

## ВҝQuГ© es un VPS?

**VPS = Virtual Private Server** (Servidor Privado Virtual)

ImagГӯnalo asГӯ:

| analogy | Description |
|--------|-------------|
| **Shared Hosting** | Es como un departamento compartido con otros vecinos - recursos limitados, depends de los vecinos |
| **VPS** | Es tu propio departamento - tienes recursos dedicados (CPU, RAM, disco) que solo tГә usas |
| **Servidor Dedicado** | Es una casa completa - todo para ti, pero mГЎs caro |

Un VPS es una computadora remota (en la nube) que puedes controlar desde tu PC. Tiene:
- **CPU**: Procesamiento
- **RAM**: Memoria trabajo
- **SSD**: Disco para archivos/DB
- **IP pГәblica**: Para que sea accesible desde internet

## Flujo Principal de una Llamada

### 1. Inicio de Llamada
- **Llamada entrante**: Telnyx envГӯa webhook `call.initiated` вҶ’ `webhooks.js` вҶ’ `answerCall()` + `startRecording()`
- **Llamada saliente**: API `/api/make-call` вҶ’ `telnyxClient.makeOutboundCall()` вҶ’ `session.service.js`

### 2. SesiГіn WebSocket (`/neural-stream`)
- `session.service.js:createSession()` maneja el flujo completo:
  1. Recibe audio Ој-law@8kHz de Telnyx
  2. EnvГӯa a Deepgram (STT) вҶ’ texto
  3. EnvГӯa texto + historial a Groq (LLM) вҶ’ respuesta
  4. EnvГӯa respuesta a ElevenLabs (TTS) вҶ’ audio Ој-law
  5. EnvГӯa audio de vuelta a Telnyx вҶ’ cliente

### 3. FinalizaciГіn
- Telnyx envГӯa `call.hangup` вҶ’ `webhooks.js`
- Guarda transcripciГіn y mГ©tricas en PostgreSQL
- Si hay `retry_interval`, programa siguiente llamada

## Componentes Clave

### Routes

| Endpoint | FunciГіn |
|----------|---------|
| `POST /api/make-call` | Crear llamada inmediatas/programada |
| `GET /api/calls` | Historial de llamadas |
| `GET/POST /api/prompts` | CRUD prompts de identidad |
| `GET/POST /api/reminders` | CRUD prompts de recordatorio |
| `GET /api/stats` | MГ©tricas de consumo |
| `POST /webhook/telnyx` | Eventos de Telnyx |
| `POST /api/tts-preview` | PrevisualizaciГіn de voz |

### Servicios AI

- **STT** (`stt.service.js`): Deepgram Nova-3, espaГұol, Ој-law@8kHz
- **Brain** (`brain.service.js`): Groq llama-3.1-8b-instant, historial conversacional
- **TTS** (`tts.service.js`): ElevenLabs turbo_v2_5, normalizaciГіn de texto

### Cola de Llamadas

- `queue.service.js`: Implementa semГЎforo para evitar saturaciГіn del canal Telnyx
- `MAX_CONCURRENT_CALLS` (default 2) controla concurrencia

### Programador

- `scheduler.service.js`: Verifica cada 60s llamadas programadas
- Recupera llamadas atoradas cada 2 min
- Soporta reintentos automГЎticos con `retry_interval_hours`

### Base de Datos

**Tablas principales**:
- `calls`: Registro de llamadas (call_id, direction, nГәmeros, dominio, transcripciГіn)
- `call_transcripts`: Turns individuales (role, text, at)
- `prompts`: Prompts de identidad
- `reminder_prompts`: Prompts de recordatorio/campaГұa
- `app_settings`: ConfiguraciГіn activa (active_prompt_id, etc.)
- `scheduled_calls`: Llamadas programadas
- `usage_logs`: MГ©tricas por servicio (tokens, segundos, caracteres)

| `/neural-stream` | Streaming de audio bidireccional |

## AutenticaciГіn

- **Dashboard**: Cookie `auth_token` con TTL 24h
- **Webhook Telnyx**: VerificaciГіn HMAC-SHA256 con `TELNYX_PUBLIC_KEY`

## Consideraciones de Seguridad

1. **Credenciales**: Todas en `.env` (no committed)
2. **Firma webhook**: Validada por defecto (`ENFORCE_TELNYX_SIGNATURE=true`)
3. **Rate limiting**: 50 llamadas/IP/minuto en `/api/make-call`
4. **Sesiones**: Tokens en memoria con expiraciГіn

## Potential Issues Observados

1. **Memoria**: `callState.inflightOutbound.clear()` cada 5 min sin distinciГіn - podrГӯa borrar llamadas legГӯtimas
2. **Errores silenciados**: `call.hangup` silenced errors para code 90018 - puede ocultar problemas
3. **Context GC**: Limitado a 200 entries - podrГӯa perder contexto en spikes
4. **Transcript storage**: Elimina y re-insert en update - posible race condition

## MГ©tricas de Consumo

- **Deepgram**: segundos de audio
- **Groq**: tokens_in + tokens_out
- **ElevenLabs**: caracteres sintetizados
- **Telnyx**: duraciГіn en segundos

## Pasos de Despliegue

### 1. PreparaciГіn del VPS (Ubuntu 22.04)

```bash
# ConexiГіn SSH
ssh root@TU_IP_VPS

# Actualizar sistema
apt update && apt upgrade -y

# Crear usuario deploy (no ejecutar como root)
adduser deploy
usermod -aG sudo deploy

# Configurar SSH keys (recomendado) y disable root login
```

### 2. Instalar Dependencias

```bash
# Node.js 20.x (via nvm)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 20
nvm alias default 20

# PostgreSQL 14+
apt install postgresql postgresql-contrib -y

# Nginx
apt install nginx -y

# Certbot (Let's Encrypt)
apt install certbot python3-certbot-nginx -y

# PM2 (process manager)
npm install -g pm2
```

### 3. Configurar PostgreSQL

```bash
# Configurar PostgreSQL
sudo -u postgres psql

# En psql:
CREATE DATABASE viaai;
CREATE USER viaai_user WITH PASSWORD 'TU_PASSWORD_SECURE';
GRANT ALL PRIVILEGES ON DATABASE viaai TO viaai_user;
\q

# Configurar autenticaciГіn
# Editar /etc/postgresql/15/main/pg_hba.conf
# Cambiar a md5 para localhost
```

### 4. Subir CГіdigo al VPS

```bash
# OpciГіn A: Git clone
cd /opt
git clone TU_REPOSITORIO viaai
cd viaai
npm install --production

# OpciГіn B: SCP
scp -r ./llamadas root@TU_IP:/opt/viaai
```

### 5. Configurar Variables de Entorno

```bash
# Crear .env en /opt/viaai/
cp .env.example .env
nano .env

# Variables crГӯticas para producciГіn:
# - DATABASE_URL=postgresql://viaai_user:PASSWORD@localhost:5432/viaai
# - WS_URL=wss://TU_DOMINIO/neural-stream
# - PORT=3000
# - ENFORCE_TELNYX_SIGNATURE=true
```

### 6. Inicializar Base de Datos

```bash
# El servidor lo hace automГЎticamente en startup
# O manualmente:
psql -U viaai_user -d viaai -f schema.sql
```

### 7. Configurar PM2

```bash
cd /opt/viaai
pm2 start server.js --name viaai
pm2 startup
pm2 save
```

### 8. Configurar Nginx como Reverse Proxy

```nginx
# /etc/nginx/sites-available/viaai
server {
    listen 80;
    server_name TU_DOMINIO;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
    }

    location /neural-stream {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }
}
```

```bash
# Activar config
ln -s /etc/nginx/sites-available/viaai /etc/nginx/sites-enabled/

systemctl reload nginx
```

### 9. SSL con Let's Encrypt

```bash

```

### 10. Firewall (UFW)
